# ПДД AI — Этап 1: каркас. Инструкция по запуску

Это рабочий скелет сайта на Next.js: главная, вход по коду на почту,
заглушки экранов (обучение, экзамен, справочник, профиль), карта-путь тем,
маскот, адаптивная вёрстка под телефон. Архитектура заложена под Supabase
и серверные функции — на следующих этапах нарастим начинку.

Делается один раз ~20-30 минут. Иди строго по шагам.

---

## Шаг 1. Настрой базу данных в Supabase

1. Зайди в свой проект на supabase.com (или создай новый: New project,
   придумай пароль БД, регион выбери поближе — Frankfurt).
2. Слева открой **SQL Editor** → **New query**.
3. Открой файл `supabase/schema.sql` из этого архива, скопируй ВСЁ
   содержимое, вставь в редактор и нажми **Run**.
   Должно появиться «Success». Это создаст все таблицы (профили, прогресс
   по темам, очередь повторения, лог ответов, симуляции) и правила
   безопасности.
4. Открой **Project Settings → API**. Тебе нужны два значения:
   - **Project URL** (вида https://xxxx.supabase.co)
   - **Project API keys → anon public** (длинный ключ)
   Скопируй оба — понадобятся в шаге 3.

### Включи вход по коду на почту
5. **Authentication → Providers → Email**: убедись, что Email включён.
   Найди опцию подтверждения — нам нужен вход по коду (OTP). В новых
   проектах Supabase это работает из коробки: при входе пользователю
   приходит письмо с кодом.
6. **Authentication → URL Configuration**: в поле **Site URL** впиши
   адрес, который даст Vercel после деплоя (вернёшься сюда в шаге 5),
   а пока можно оставить http://localhost:3000.

---

## Шаг 2. Залей проект на Vercel

Вариант проще — через интерфейс Vercel:
1. Распакуй архив. Залей папку проекта в репозиторий на GitHub
   (или используй Vercel CLI — см. ниже).
2. На vercel.com → **Add New → Project** → подключи этот репозиторий.
3. Vercel сам определит, что это Next.js. НЕ нажимай Deploy сразу —
   сначала добавь переменные окружения (шаг 3).

Вариант без GitHub (через CLI):
1. Установи: `npm i -g vercel`
2. В папке проекта: `vercel` — следуй подсказкам, привяжи к аккаунту.

---

## Шаг 3. Добавь переменные окружения в Vercel

В проекте на Vercel: **Settings → Environment Variables**. Добавь две:

| Имя | Значение |
|-----|----------|
| `NEXT_PUBLIC_SUPABASE_URL` | Project URL из шага 1 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon public ключ из шага 1 |

Сохрани. (Локально для теста — скопируй `.env.local.example`
в `.env.local` и впиши туда те же значения, затем `npm run dev`.)

---

## Шаг 4. Запусти деплой

Нажми **Deploy** на Vercel. Через минуту-две получишь живой адрес
вида `https://pdd-ai-xxx.vercel.app`. Открой — увидишь главную страницу
с маскотом и кнопками.

---

## Шаг 5. Свяжи адрес с Supabase (важно для входа)

1. Скопируй адрес, который дал Vercel.
2. В Supabase → **Authentication → URL Configuration**:
   - **Site URL**: вставь адрес Vercel.
   - **Redirect URLs**: добавь `https://твой-адрес.vercel.app/auth/callback`
3. Сохрани. Теперь вход по коду на почту заработает на живом сайте.

Проверь: открой сайт → Войти → введи свою почту → получишь код в письме →
введи код → попадёшь на экран обучения. Готово.

---

## Что работает после Этапа 1
- Главная страница, адаптив под телефон.
- Вход по коду на почту (Supabase Auth).
- Навигация по экранам-заглушкам.
- База данных создана и ждёт наполнения.

## Чего пока нет (следующие этапы)
- Этап 2: 800 вопросов, движок адаптации, режим обучения, работа над ошибками.
- Этап 3: симуляция экзамена, свободное тестирование.
- Этап 4: уроки и справочник знаков.
- Этап 5: геймификация (карта-путь, уровни, маскот, готовность).
- Этап 6: оплата, серверный прокси для AI-репетитора.

## Вход через Google и Яндекс (можно позже)
Кнопки уже есть на странице входа, но чтобы они заработали, нужно
завести приложения в консолях Google и Яндекс и вставить их ключи
в Supabase (Authentication → Providers). Дам отдельную инструкцию,
когда дойдём — на работу остальных функций это не влияет.

---

## Если что-то не так
- Сайт собрался, но вход не работает → проверь Redirect URLs в Supabase
  (шаг 5) и что переменные окружения в Vercel заданы без пробелов.
- Письмо с кодом не приходит → проверь папку спам; в Supabase на
  бесплатном тарифе есть лимит писем, для теста хватает.
- Ошибка сборки на Vercel → убедись, что залит весь проект, включая
  package.json, и переменные окружения добавлены до деплоя.
